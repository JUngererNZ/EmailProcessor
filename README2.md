## Overview

This script **analyzes a directory structure** for FML Freight Solutions projects and generates automated reports. It scans files, extracts metadata and reference data from filenames, and produces two outputs: a JSON data file and a populated markdown checklist.

## How It Works

**Main Class: `DirectoryAnalyzer`**

1. **Directory Scanning** (`analyze_directory()`)
   - Walks through all files and folders
   - Collects file metadata: size, modification date, extension, paths
   - Counts file types

2. **Data Extraction** (`extract_reference_data()`)
   - Uses regex patterns to find shipping info in filenames:
     - PIN numbers (6-8 digits)
     - Serial numbers, Engine numbers, Vessel/Voyage info
     - Bill numbers, Container numbers

3. **Analysis Methods**
   - `analyze_billing_rates()` — Detects invoices, POs, quotations, weight/cube data
   - `analyze_tasks_progress()` — Marks task completion based on document presence (logistics, customs, delivery, financials)
   - `generate_visuals_verification()` — Checks for WE/XE pictures

4. **Output Generation**
   - `save_json_output()` — Saves structured analysis as JSON
   - `save_checklist()` — Generates a formatted markdown checklist with status indicators

## Usage

Run with a directory path:
```bash
python directory_analyzer.py /path/to/directory
```

Or interactively (prompts for input). Creates:
- directory_analysis.json — Raw data with statistics, file lists, extracted reference data
- `checklist.md` — Formatted project checklist with shipping reference info, billing status, task progress, and action items

The script is designed to auto-populate shipping documentation checklists by analyzing file names and document presence.
