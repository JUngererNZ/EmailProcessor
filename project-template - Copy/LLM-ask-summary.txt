**Situation**
You have multiple markdown and PDF files containing information about events, tasks, or activities that are scattered across different files and may not be in chronological order.

**Task**
The assistant should extract all relevant information from the provided markdown and PDF files, identify dates and timestamps, and compile comprehensive notes organized in strict chronological order from earliest to latest into a single markdown document.

**Objective**
Create a unified timeline document that presents all information in chronological sequence with consistent formatting, making it easy to understand the progression of events or activities across all source materials.

**Knowledge**
- Extract all dates, times, and temporal references from the markdown and PDF files
- When exact dates are missing, use contextual clues to infer relative ordering
- Preserve all relevant details from the original files (descriptions, context, metadata)
- If multiple events occur on the same date, order them by time of day if available; otherwise, maintain logical sequence
- Consolidate duplicate or related entries under the same timeline point
- Flag any ambiguous or unclear dates for clarification
- Include the source file name for each entry to maintain traceability
- For PDF files, extract text content and treat it with the same priority as markdown files

**Output Format Requirements**
- Format the output as a single markdown file with date-based section headings (YYYY-MM-DD format)
- Each timeline entry should include:
  - **Date**: Exact date in YYYY-MM-DD format
  - **Time**: Exact timestamp if available (HH:MM format)
  - **Event Title**: Concise heading for the event
  - **Summary**: 2-3 sentence description of what occurred
  - **People Involved**: Names or roles of individuals mentioned
  - **Action Items**: Any tasks, follow-ups, or next steps (if applicable)
  - **Source**: Original file name for traceability
- Use bullet points for people involved and action items
- Arrange all entries in strict chronological order from earliest to latest date
- Use consistent date formatting throughout (YYYY-MM-DD)
- Include time information when available, ordered by time of day within the same date
